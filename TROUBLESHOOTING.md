# Устранение проблем с определением доступности приложений

## Проблема
Скрипт помечает живые приложения как "мертвые" (недоступные).

## Причины и решения

### 1. Улучшенная логика определения доступности

Мы исправили логику в `checker.py`:

- **Более точные паттерны "not found"** - убрали слишком широкие паттерны
- **Терпимость к минимальным метаданным** - приложение считается доступным, если нет явных признаков недоступности
- **Проверка индикаторов страницы приложения** - даже без метаданных, если страница выглядит как страница приложения, считаем доступным

### 2. Улучшенный поиск скриншотов и модального контента

- **Поиск по стабильным паттернам** - вместо динамических классов используем `role="button"`, `aria-label`, `data-*` атрибуты
- **Множественные стратегии поиска** - ищем скриншоты по различным контейнерам и структурам
- **Поиск модального контента** - ищем расширенную информацию в диалогах и разворачиваемых секциях
- **Адаптивные паттерны** - автоматически подстраиваемся под различные форматы страниц Google Play

### 3. Улучшенная обработка ошибок

- **Повторные попытки** - автоматические повторные попытки при временных ошибках сети
- **Умная классификация ошибок** - различаем временные (сеть) и постоянные (приложение недоступно) проблемы
- **Сохранение статуса** - при временных ошибках статус приложения не меняется на "недоступно"
- **Подробное логирование** - детальная информация о причинах ошибок для отладки

### 4. Новые команды для отладки

#### `/test_check <ссылка>`
Тестирует проверку приложения без обновления базы данных. Полезно для отладки.

#### `/test_modal <ссылка>`
Тестирует поиск модального контента и расширенной информации. Полезно для проверки извлечения скриншотов и детального описания.

#### `/test_error_handling <ссылка>`
Тестирует обработку ошибок и показывает, как временные проблемы с сетью обрабатываются системой.

#### `/debug <ссылка>`
Показывает текущий статус приложения в базе данных.

### 5. Тестовый скрипт

Запустите `test_checker.py` для тестирования логики:

```bash
python test_checker.py
```

### 6. Логирование

Включено подробное логирование для отладки. Проверьте логи при возникновении проблем.

## Понимание сообщений об ошибках

### Временные ошибки (не влияют на статус приложения):
- `timeout` - превышение времени ожидания
- `connection` - проблемы с подключением
- `request` - общие ошибки запроса
- `unexpected` - неожиданные ошибки

### Постоянные ошибки (приложение недоступно):
- HTTP 404, 410, 451 - приложение не найдено или заблокировано
- Явные сообщения "not found" в HTML

**Важно**: Временные ошибки НЕ помечают приложение как недоступное. Статус сохраняется до следующей успешной проверки.
